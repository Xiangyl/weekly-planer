# 📋 周计划 · 每日任务

一个轻量级的个人周计划管理工具，帮助你有条不紊地规划每周任务并追踪每日完成情况。

## ✨ 功能

- **📝 周计划管理** — 创建可复用的周计划模板，预设每周要完成的任务
- **📅 每日任务挑选** — 每天从周计划任务池中挑选当天想完成的任务
- **✅ 完成追踪** — 勾选完成任务，实时显示完成进度
- **📊 历史回顾** — 按周查看过去的每日完成情况，含完成率统计图
- **💭 每日反思** — 每天记录心得感悟
- **🎯 临时任务** — 支持快速添加不在计划中的临时任务
- **🌙 深色模式** — 支持明暗主题切换
- **💾 数据备份** — 支持 JSON 格式导出/导入数据

## 🚀 使用方式

### 方式一：直接打开（推荐）

直接用浏览器打开 `index.html` 文件即可使用。数据存储在浏览器的 localStorage 中。

### 方式二：部署到静态服务器

将 `index.html` 部署到任意静态文件服务器，如：

- **GitHub Pages** — 免费，推送即部署
- **Netlify** — 拖拽文件即可上线
- **Vercel** — 支持单文件部署

### 方式三：本地开发

```bash
# 用任意 HTTP 服务器启动
npx serve .
# 或
python -m http.server 8080
```

## 📖 使用教程

### 1. 创建周计划

切换到「📝 周计划」标签，创建一个新的计划模板，添加你想要每周完成的任务。可以为每个任务设置：
- **优先级**：高/中/低
- **星期提示**：点击星期标签设置建议哪天完成（可选）

### 2. 应用到本周

在「📅 今日」页面，选择一个周计划模板，点击「应用到本周」。所有任务会进入本周任务池。设置了当天星期提示的任务会自动出现在今日列表中。

### 3. 每日使用

每天打开应用：
1. 查看今日任务列表
2. 从右侧任务池中点击 `+` 添加想做的任务
3. 勾选完成的任务
4. 随时添加临时任务
5. 写下每日反思

### 4. 查看历史

切换到「📊 历史」标签，回顾过去的完成情况。

## 🛠 技术栈

- **Alpine.js** — 轻量级响应式框架
- **Alpine Persist** — 自动同步 localStorage
- **原生 CSS** — CSS Variables 主题系统
- **Canvas API** — 历史统计图
- **零依赖构建** — 单文件 HTML，无需 npm/build

## 📦 数据存储

所有数据存储在浏览器的 `localStorage` 中，key 为 `wwp:v1:state`。数据包括：
- 周计划模板
- 每周任务实例
- 每日完成记录

**重要**：清除浏览器数据会导致记录丢失，请定期导出备份。
